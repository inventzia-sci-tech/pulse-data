# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in pulse-data, please report it
**privately** by email to:

    operations@inventzia.com

Please include:

- A description of the vulnerability.
- Steps to reproduce, or a minimal proof of concept.
- The version, branch, or commit affected.
- Any relevant logs (please avoid posting full exploit code in this report).

We will acknowledge receipt within **5 business days** and aim to share a
remediation plan within **30 days** for confirmed issues.

Please **do not** open public GitHub issues for security vulnerabilities; that
exposes other users before a fix is available.

## Supported versions

Until pulse-data reaches its first stable release, security fixes are applied
only to the `main` branch. Once stable releases are tagged, this section will
list the version ranges that receive backports.
