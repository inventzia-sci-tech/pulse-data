## Generating schemas

### Python

From the repository root:

```bash
python schemas/schemas-generators/autogenerate_schemas.py \
  --schemas-dir schemas/schemas_yaml \
  --output-dir schemas/schemas_py \
  --dry-run -v
```

Remove `--dry-run` to write files. The generator walks `schemas_yaml/` recursively, converts
each YAML definition to a Pydantic model, and writes one `.py` file per schema into the
mirrored path under `schemas_py/`.

`autogenerate_schemas_with_fields.py` is an experimental variant that emits explicit field
declarations rather than dynamic model construction. Use `autogenerate_schemas.py` as the
canonical generator until the field variant is promoted.

### Java

Java classes are generated by the `jsonschema2pojo` Maven plugin configured in the root
`pom.xml`. To run it:

- **IntelliJ:** open the Maven panel (View → Tool Windows → Maven), expand
  Plugins → jsonschema2pojo, and double-click `jsonschema2pojo:generate`.
- **Command line:**
  ```bash
  mvn jsonschema2pojo:generate
  ```

Generated classes are written to `schemas/schemas_java/`. Do not edit them by hand.

### After generation

- Regenerate both sides whenever a YAML schema changes.
- The generated Java class must implement `com.inventzia.beacon.core.Event` and declare
  `public static final String TYPE_ID` equal to the schema `$id`. This is not yet enforced
  by the generator — it must be added manually or via a post-generation hook until codegen
  support for the `Event` interface is wired in.
